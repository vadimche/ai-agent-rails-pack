# Trust and Safety Notes

This project is a workflow pack, not an automated assurance product.

## What it helps with

- making intent explicit before coding;
- reducing accidental scope drift;
- making invariants visible;
- improving human review handoff;
- keeping agent sessions smaller and more structured.

## What it does not guarantee

- code correctness;
- production safety;
- security;
- regulatory compliance;
- architectural quality;
- legal or professional advice.

## Recommended usage

Use this pack as a human-controlled workflow layer. Keep approval gates explicit. Do not let an agent publish, deploy, migrate data, rotate credentials, change payment logic, or modify security-sensitive code without human review.

## Safe language for teams

Use:

> This pack helps structure AI-assisted development.

Do not use:

> This pack guarantees safe AI-generated code.
